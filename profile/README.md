# PageHelper Project Account Migration Update
Dear Users,

We are currently upgrading the original personal account `pagehelper` to an organization with the same name (`pagehelper`). This operation is in response to the requirements of Open Source Collective (OSC)'s fiscal hosting application — OSC stipulates that only projects hosted under GitHub organizational accounts are accepted, to ensure shared responsibility for the project and enhanced access control security, thereby facilitating the smooth progress of the fiscal hosting application. However, during the migration of our two core projects, `Mybatis-PageHelper` and `pagehelper-spring-boot`, to the new organization, we encountered a technical issue related to GitHub repository name occupancy. We have contacted GitHub official support to coordinate a resolution and will update you on the progress subsequently.

### Temporary Access Methods
Until the migration is completed, you can access the original projects via the following temporary URLs. All operations (cloning, committing, pull requests, Issues, etc.) remain unaffected and can be used normally:
1. Mybatis-PageHelper: https://github.com/ruiqianyi/Mybatis-PageHelper
2. pagehelper-spring-boot: https://github.com/ruiqianyi/pagehelper-spring-boot

### Follow-up Instructions
Once the GitHub issue is resolved and the project migration is finalized, the original access addresses (`pagehelper/Mybatis-PageHelper` and `pagehelper/pagehelper-spring-boot`) will be restored. We will notify you immediately upon completion. Please rest assured that the migration process will not impact project data or historical commit records.

If you have any questions, please leave a comment via the project Issues. Thank you for your understanding and support!
